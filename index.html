Instructions

Operations and constants are case-insensitive.
Variables are case sensitive, can be longer than a single character, can only contain alphanumeric characters, digits and the underscore character, and cannot begin with a digit.

Highest precedence operations

These operations are evaluated first, from left to right. To execute them a particular operation first, surround it with parenthesis ( ).

OperationAliasesDescription
NOT A ~ ' ! ¬ false if A is true; true if A is false.
A AND B ∧ ^ & · * true if both A and B are true; false if either or both are false.
A EQUIV B ⇔ ≡ ↔ = true if both A and B are true or both are false; false if one is true and the other is false.
A IMPLIES B ⇒ → ⊃ > true if A is false, or A is true and B is true; false if A is true and B is false.

Lower precedence operations

These operations are evaluated after the ones above, from left to right. To execute them first, surround them with parenthesis ( ).

OperationAliasesDescription
A OR B ∨ v + ∥ | true if A is true, B is true, or both are true; false if either or both are false.
A XOR B ⊻ ⊕ true if A is true and B is false; true if A is false and B is true; false if both are true or both are false.

Constants

ConstantAliasesDescription
TRUE 1 T ⊤ true Always evaluates to true.
FALSE 0 F ⊥ false Always evaluates to false.
